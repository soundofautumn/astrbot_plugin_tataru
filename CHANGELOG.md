# 更新日志

## v0.14.21

### 新增

- 新增 `logs 角色名 服务器名` 命令，通过 FFLogs v2 GraphQL Character API 查询角色公开战绩。
- `logs` 支持角色名带空格，服务器名放在最后，命令末尾可追加 `国服` / `国际服` 临时覆盖默认 FFLogs 站点。
- `logs` 默认返回绝境战、7.0 阿卡狄亚零式、6.0 万魔殿零式的百分位、职业、rDPS、排名和角色页链接。

## v0.14.20

### 新增

- `输出` 支持阿卡狄亚登天斗技场 M1-M12 普通难度和 M1S-M12S 零式缩写检索，并补充轻量级/中量级/巡航级/重量级中文别名。
- `输出` 支持多词 boss 名称解析，例如 `阿卡狄亚零式登天斗技场 轻量级1 舞者`。
- `输出` 查询 `M12S` / `林德布鲁姆` 时默认同时返回林德布鲁姆和林德布鲁姆 II，指定 `门神`/`本体` 时只返回对应阶段。

## v0.14.19

### 修复

- `输出` 默认查询改为直接请求 Chrome DevTools 确认的 FFLogs statistics XHR 路径：`100/1/14/0/.../normalized/.../?keystone=15&dpstype=...`。
- `输出` 保留页面解析诊断作为兜底，但优先使用浏览器同款 statistics table 响应，避免旧 `100/1000/7/0` 时间序列口径。

## v0.14.18

### 调试

- `输出` 增加 FFLogs 初始 HTML 中脚本 src、疑似 statistics/api URL、table 标签摘要和关键字符串计数日志，用于定位浏览器最终 DOM 的真实数据来源。

## v0.14.17

### 修复

- `输出` 在 FFLogs 初始 HTML 没有最终 DOM class 时，会从页面全局小数候选中提取一组按 100/99/95/75/50/25/10 递减的秒伤值。
- `输出` 增加全局秒伤候选和递减数值块诊断日志，用于继续确认 FFLogs 前端渲染数据结构。

## v0.14.16

### 调试

- `输出` 默认查询暂时停用 FFLogs `statistics/table` 时间序列兜底，statistics 页面解析失败时直接返回失败提示，避免输出错误口径数据。
- `输出` 增加 FFLogs 页面表格诊断日志，记录 `td`/`primary`/`main-table-number` 命中数量、候选秒伤样例和职业附近候选值，便于继续定位页面 DOM。

## v0.14.15

### 修复

- `输出` 解析 FFLogs statistics 页面时优先读取底部表格中的 `td.main-table-number.primary` 秒伤单元格，匹配网页当前实际 DOM 结构。
- `输出` 命中底部表格主数值时会记录 `FFLogs statistics primary table cell matched`，便于确认不再回退到时间序列接口。

## v0.14.14

### 修复

- `输出` 解析 FFLogs statistics 页面底部职业表时不再强制要求日期范围，避免页面中表格有数据但日期由前端单独渲染时直接回退到时间序列接口。
- `输出` 的底部表格解析仍要求秒伤候选值大于等于 1000，避免再次误取版本号或菜单数字。

## v0.14.13

### 修复

- `输出` 在 FFLogs statistics 页面含有图表标签但标签和值分离时，会从图表配置附近提取一组按 100/99/95/75/50/25/10 递减的秒伤值。
- `输出` 的图表数值块解析同时支持国服中文标签和国际服英文标签。

## v0.14.12

### 调试

- `输出` 在 FFLogs statistics 页面解析失败时记录安全诊断信息，只包含页面长度和关键标记，不输出完整 HTML、Cookie 或账号信息。

## v0.14.11

### 修复

- `输出` 解析 FFLogs statistics 页面时优先从柱状图标签提取 10/25/50/75/95/99/100 百分位，避免把页面版本号等菜单数字误判为秒伤。
- `输出` 的底部表格兜底解析增加日期范围和数值下限校验，避免返回 `7.00` 这类无效结果。

## v0.14.10

### 修复

- `输出` 默认改为按 `dataset=100/99/95/75/50/25/10` 抓取 FFLogs statistics 页面底部职业表的秒伤数据，使结果与网页柱状图和底部表格保持一致。
- 保留原 statistics table 时间序列接口作为指定 `dayN` 或页面解析失败时的兜底。

## v0.14.9

### 修复

- `输出` 查询 FFLogs statistics table 时优先使用 `persecondamount`，更贴近网页默认展示的 DPS 百分位。
- 移除 FFLogs statistics table 请求中与 FF14 统计页无关的 `keystone=15` 参数，避免额外筛选影响结果。

## v0.14.8

### 修复

- 修正 FFLogs statistics table 路径中的副本人数段，恢复为 `8`，避免误用网页 `bracket` 参数导致 table 无数据。

## v0.14.7

### 修复

- `输出` 查询 FFLogs statistics table 时按多个聚合路径依次尝试，并在 `amount` 不可用时回退到 `normalized`，避免直接返回无数据。
- `输出` 日志会记录命中的 statistics 聚合路径，便于继续和 FFLogs 网页实际请求对齐。

## v0.14.6

### 修复

- 移除 `输出` 的 `dataset` / `bracket` 命令参数支持；`dataset` 是页面当前选中的百分位，不再作为 table 全局查询参数传递。
- `输出` 查询 FFLogs statistics table 时记录实际请求 URL，便于和网页请求继续对照。

## v0.14.5

### 修复

- `输出` 支持在命令中追加 `dataset99` / `dataset=99`、`bracket6` / `bracket=6` 覆盖 statistics table 参数。

## v0.14.4

### 修复

- `输出` 的 FFLogs statistics table 聚合口径从 `normalized` 改为 `amount`，匹配网页统计表的 DPS / Max 数值口径。

## v0.14.3

### 修复

- `输出` 默认 DPS 类型改为 rDPS，匹配 FFLogs statistics 默认口径。
- `输出` 查询 rDPS 时不再向 FFLogs statistics table 拼接 `dpstype` 参数；仅在显式指定 `adps`、`pdps`、`ndps`、`cdps` 时传递该参数。
- 修正 FFLogs 重复分区选择：国际服选择每组第 1 个分区，国服选择每组第 2 个分区，避免误用韩服分区。
